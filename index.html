Naming convention for filenames, paths and folders

Filenames, paths and folders must be named using only lowercase letters, no spaces, and no special characters (only letters, numbers and dashes).

Good examples: reference-website, double-layer-chocolate-cake.jpg, main.css

Best Practices for Git Commit Messages

Commit message best practices:

  • Minimum 3 words & 10 characters
  • Proper spelling & grammar
  • Capital starting letter, no period
  • Must start with an imperative verb: Create, Fix, Add, Solve, etc.
  • Pretend every commit starts with the phrase: “This commit will…”

Commonly used HMTL tags/elements

Headings

The heading tags are used to mark headings according to their importance. The h1 tag stands for the most important heading of the web page and the h6 stands for the least important and smallest one.

  1. H1

  2. H2

  3. H3

  4. H4

  5. H5
  6. H6

Paragraph Tag

Defines a paragraph of text. It is a block-level element and always starts on a new line. Before and after each paragraph,browsers add margin automatically.

Unordered List

Use unordered lists for lists of items that can appear in any order, for example, a shopping list

  • cheese
  • apples
  • milk

Ordered Lists

Use OL tags for lists that follow a specific order, for example, the chronological steps of a recipe

  1. Step one
  2. Step two
  3. Step three

Description List

A description list includes a term, and the corresponding description that accompanies it. For example, a dictionary term, and it's definition would be marked up using a description list

Noun
person, place or thing
Verb
describes an action

anchor tag

Anchor tags create a clickable link. The opening tag tells where the link should start and the closing tag indicates where the link ends. To create a hyperlink, you should use the anchor (a) tag and href attribute, the value of which is the URL, or location, where the link is pointing to. Always include an aria-label that provides a description of where the link goes to, for a better user experience screen reader users.

Click Here

image tag

Images in websites are not embedded in the website, but linked to by the HTML document. Images need two attributes to specify extra details about the image: the location of the image and the alternative content. src is a path to the image file. alt describes the image if it cannot be seen

figure, figcaption

figure tags contain self-contained content, like illustrations and diagrams, with a caption. An image tag linking to the content is nested within a figure tag to display the item. The figure caption tag is also nested within the figure tag. Figure caption cannot stand on it's own.

q

the q tag is used for a short, inline quotation

The weather was finally starting to become warm, the students all said Welcome spring in unison.

blockquote

blockquote tag is used for stand alone quotations

This is my reference website!

cite

The cite tag references an author, or source for a quote.

Kristen Scheel

em

A string of emphasized, slightly more important text. Screen readers will read this text in a different voice.

Assignments are typically due at the beginning of each week. The reference website is due Monday, April 3

strong

a strong tag adds lots of emphasis to the content inside. screenreaders will read this content is a different voice

The penalty for unpaid parking is a parking ticket. After three violations your vehicle will be towed

b

b defines a keyword, like a product name in a review, or a lead sentence in a paragraph. The b tag is used to markup text as bold without conveying any extra importance. It will not be read in a different voice by screenreaders.

This is a paragraph This section of text is bold, but it is not more important.

i

The i tag sets the content in italics. This can be used when defining a technical term, listing a book title, or setting text that needs to be emphasized subtly (sarcasm for example)

I read the latest review in today's New York Times

small

the small tag is used for fine print, for example, for copyright info in the footer

copyright 2023

Structuring Content

header

The header is the introductory content usually at the top of the page and would include a logo or company name and main navigation. Headers are also within sections and articles to introduce the content for headings, subheadings, author, etc. All content for the header goes in between the opening and closing header tags

main

The main element wraps around the main content of the page. It is only used once per page and it tells the browser that this is where the main content is.

footer

Many webpages end their pages with a footer at the bottom. This often includes lists of links, copyright information, extra information about the company, terms and services.

All content for the footer goes in between the opening and closing footer tags

section

Section is used to wrap around related pieces of content. Usually each section has its own heading.

article

The article element represents a self- contained composition in a web page, which is intended to be independently distributable or reusable. An article could be removed from the website, and would still make senese on it's own. Examples include blog post, product card, interactive widget, etc...

nav

The nav element represents a section of a page whose purpose is to provide navigation links, either within the current document or to other documents.

Font properties

When styling fonts in CSS there are a number of properties that we can use to control the style.

font-family

font family is where we specify the name of the font (i.e. Helvetica, or, sans-serif)

font-size

The font-size property is used to set the size of the font. It is a single number with a unit (rem, em, or px - but rem is the best choice, so that the site can be responsive, and respect users desired settings)

font-weight

font weight sets the thickness (or thinness) of the characters, for example, bold, normal, 500, 300, 700.

font-style

Specifies the font style, for example, italic or normal.

line-height

this property is used to adjust the line height (the space between lines of text). It should be a single number, a multiplier that multiplies the font size (ex. 1.5). Does not take a unit!

font shorthand

To shorten the code, it is possible to specify more than one individual font property in one declaration

Text decorations

Text decorations are changes that can be added to text in CSS, for example, text transform, text-align, text-indent, text-shadow, text-decoration. Examples below.

text-transform

text-transform is used in CSS to change capitalizations, such as uppercase, lowercase, capitalize. It is preferable to use text-transform to make copy all caps, instead of writing it in all caps in HTML for accessibility

text-decoration

Text decoration specifies the changs to text, i.e. underline, line-through, overline, none. Only links should be underlined on a website.

text-align

Text align specifies the horizontal alignment of text inside an element. For example, left, center, right, justify.

text-indent

indents the first line of text, similar to printed materials. for example: left, center, right, justify.

text-shadow

adds a shadow to the text. Specified in this way: text-shadow: horizontal-offset vertical-offset blur-radius color. ie. text-shadow: 2px 2px #ff0000; Use sparingly!

Styling Links

Styling links in websites is no different from styling other elements, but, they must be styled directly, and don't inherit certain properties from their parent elements. Links have many different states that we can style when a user interacts. It is important that these link states are styled in the proper direction (link, visited, hover, focus, active). Links can be styled with any CSS property (color, font-family, background, etc).

link

:link is used to style the default link state. Be careful when removing underlining from a link! Make sure there is still enough contrast to understand that the text contains a link.

visited link

:visited is used for styling the link after it has been clicked on (visited) by the user

hover

:hover changes the styling for the link when the user hovers over it with the mouse

focus

:focus styles the link when the keyboard has tab-focused on it

active

:active styles the link when the mouse button is pressed down on it

Margin

outside the box is the margin, a transparent layer pushing other boxes away. Margin is the space around an element that separates it from other elements. We can specifiy margin with individual properties (i.e. margin-top, margin-left etc, but it is often easier to use the shorthand version. Start at the top and make your way clockwise around the box (i.e. margin: (30px 40px 30px 40px);

Padding

padding sits around the outside of the content, pushing the edge of the box away from the content. Padding is often used to give the content some breathing room. We can specifiy padding with individual properties (i.e. padding-top, padding-left etc, but it is often easier to use the shorthand version. Start at the top and make your way clockwise around the box (i.e. padding: (30px 40px 30px 40px); In the example blue, the blue box helps to demonstrate where the padding is - outside the content, but up to the edge of the border.
