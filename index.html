Naming convention for filenames, paths and folders

Filenames, paths and folders must be named using only lowercase letters, no spaces, and no special characters (only letters, numbers and dashes).

Good examples:

  • reference-website
  • double-layer-chocolate-cake.jpg
  • main.css

Best Practices for Git Commit Messages

Commit message best practices:

  • Minimum 3 words & 10 characters
  • Proper spelling & grammar
  • Capital starting letter, no period
  • Must start with an imperative verb: Create, Fix, Add, Solve, etc.
  • Pretend every commit starts with the phrase: “This commit will…”

Commonly used HMTL tags/elements

Headings

The heading tags are used to mark headings according to their importance. The h1 tag stands for the most important heading of the web page and the h6 stands for the least important and smallest one.

  1. H1

  2. H2

  3. H3

  4. H4

  5. H5
  6. H6

Paragraph Tag

Defines a paragraph of text. It is a block-level element and always starts on a new line. Before and after each paragraph,browsers add margin automatically.

Unordered List

Use unordered lists for lists of items that can appear in any order, for example, a shopping list

  • cheese
  • apples
  • milk

Ordered Lists

Use OL tags for lists that follow a specific order, for example, the chronological steps of a recipe

  1. Step one
  2. Step two
  3. Step three

Description List

A description list includes a term, and the corresponding description that accompanies it. For example, a dictionary term, and it's definition would be marked up using a description list

anchor tag

Anchor tags create a clickable link. The opening tag tells where the link should start and the closing tag indicates where the link ends. To create a hyperlink, you should use the anchor (a) tag and href attribute, the value of which is the URL, or location, where the link is pointing to. Always include an aria-label that provides a description of where the link goes to, for a better user experience screen reader users.

image tag

Images in websites are not embedded in the website, but linked to by the HTML document. Images need two attributes to specify extra details about the image: the location of the image and the alternative content.

figure (img & figcaption) q blockquote cite ems strong b i small

Structuring Content

header

The header is the introductory content usually at the top of the page and would include a logo or company name and main navigation. Headers are also within sections and articles to introduce the content for headings, subheadings, author, etc...

All content for the header goes in between the opening and closing header tags

footer

Many webpages end their pages with a footer at the bottom. This often includes lists of links, copyright information, extra information about the company, terms and services.

All content for the footer goes in between the opening and closing footer tags

main

The main element wraps around the main content of the page. It’s only used once per page and it tells the browser that this is where the main content is.

section

Section is used to wrap around related pieces of content. Usually each section has its own heading.

article

The article element represents a self- contained composition in a web page, which is intended to be independently distributable or reusable. Examples include blog post, product card, interactive widget, etc...

nav

The nav element represents a section of a page whose purpose is to provide navigation links, either within the current document or to other documents.

Example:

  • About Us
  • Our Work
  • Contact
